## U-Boot menu runs on Flipper One screen

The U-Boot menu now works directly on Flipper One, without the need to plug an external display. You can choose different kernels and systemd targets before the Linux kernel boots.

::embed[]{url="https://www.youtube.com/watch?v=oI6Vl_uoAwI"}

Our [patched U-Boot](https://github.com/flipperdevices/u-boot) is based on a slightly modified U-Boot with LCD and button drivers.
In the release version of Flipper OS, the boot menu will likely be drawn by a small program compiled into `initramfs` and executed after the kernel starts.

The cold boot time (from the CPU power-on to a working menu) still has room for improvement:

```console
$ tio --timestamp --timestamp-format 24hour-delta  -b 1500000 /dev/tty.usbmodemflip_one_debug2
...
[00:00:02.788] I/TC: OP-TEE version: 51d99cd (gcc version 14.2.0 (Debian 14.2.0-19)) #1 Wed Jun 24 10:30:06 UTC 2026 aarch64
...
```
See the [full boot log](https://pastebin.com/raw/A0RUrdRm).

## Multiple tests of the Wi-Fi chipset

We chose the MediaTek MT7921AU Wi-Fi chipset for Flipper One. On paper, it operates on 2.4/5/6 GHz bands, can be a station and an access point, supports monitor mode and packet injection, and has an open-source driver in the mainline Linux kernel. It looked like a great hacker-friendly chipset, but then the actual tests came in.

![MT7921AU Wi-Fi chip on the Flipper One board](/files/pics/dev-log/4/wifi-chip-mt7921au.jpg)

We tested the chip and got the following results:
- :white_check_mark: *Station mode (STA)*.
- :white_check_mark: *Access Point mode (AP)*.
- :white_check_mark: *AP+STA Single-Channel Concurrency (SCC)* [(see report)](../testing/WiFi-AP-STA-SCC.md).
- :white_check_mark: *Monitor mode* [(see report)](../testing/WiFi-Monitor.md).
- *Dual-band dual-concurrent (DBDC)*:
  - :x: *Dual-band AP+STA* [(see report)](../testing/WiFi-DBDC.md#test-log-apsta-dual-band).
  - :white_check_mark: *Dual-band AP+STA with a driver patch*. Our contributor _@germiBest_ made tests with a simple driver patch and proved that dual-band AP+STA can be fixed [(see report)](https://github.com/flipperdevices/flipperone-docs/pull/387).
  - :x: *Dual-band AP+AP* [(see report)](../testing/WiFi-DBDC.md#test-log-apap-dual-band). This combination wasn't expected to work, and we made a report to prove that.
- Operating on :white_check_mark: 2.4 / :white_check_mark: 5 / :x: 6 GHz. Even configured for a region with 6 GHz enabled, the chip didn't allow us to use this band.

Last week, [we asked for your help with Wi-Fi testing](https://x.com/Flipper_RND/status/2069083164190929111) — thanks for your research and ideas. We've also contacted MediaTek about the issue with dual-band AP+STA, but got no response so far. Besides, there are [several driver patches](https://lore.kernel.org/linux-wireless/?q=build+iface+combinations+dynamically) coming to the upstream — although, we can't guarantee they will succeed soon. Wrapping it all up, we can expect that dual-band AP+STA will be enabled, but dual-band AP+AP won't ever work on this chip.

:::hint{type="info"}
So, with all the confirmed and expected results, do you think it's a good choice of Wi-Fi chip? Will it still do the job for you? Please share what you think in the comments below.
:::

## Flipper One Lab for UI testing — the 1st community project

*@kalicyh* built the [Flipper One Lab](https://github.com/kalicyh/flipperone-lab) — a container-based emulator of Flipper One's FlipCTL UI right in a web browser, no physical device needed. Check out this demo (and note that it's a browser tab!):

::embed[]{url="https://www.youtube.com/watch?v=6HPHjbBp00M"}

This is a great example of a Flipper One [community project](../community/community.md). If you work on something similar, let us know in the comments below!

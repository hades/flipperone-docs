This is our very first Dev Log where we share how the development of Flipper One is going. Updates are grouped by sub-projects, with links, screenshots, and videos so you can explore the details.

---

## 🐧 Linux (CPU Software)

- Hardware video decoding now works on RK3576 in the mainline kernel! Green artifacts still appear in some cases, so debugging continues, but this is a huge step forward. Here's a demo:

  ::embed[]{url="https://www.youtube.com/watch?v=K7k6MUypnC8"}


- Added [hardware overscan compensation](https://github.com/flipperdevices/flipper-linux-kernel/commit/7e47e5cb38d029534356bbb7ad543ce673d516c7) for HDMI output to a TV.

  ![Overscan setting](/files/pics/dev-log/overscan-setting.png "Overscan compensation setting for HDMI output")

- Audio codec (NAU88C22) power sequence is implemented in its upstream driver ([patch series](https://lore.kernel.org/all/20260525-nau8822-reg-v2-0-7d37ae393e46@flipper.net/)).

- Flipper One display and button drivers added to U-Boot, and boot menu rendering patched so it scales to the small screen (commits: [f668e48](https://github.com/flipperdevices/u-boot/commit/f668e484d0ff24c939b806dfe71969bdfd9c67b9), [e073674](https://github.com/flipperdevices/u-boot/commit/e0736740f6f06ae0422e04457f06bb7fff933936), [93d9bf9](https://github.com/flipperdevices/u-boot/commit/93d9bf95d51c1feacbc63d0690059cb5fcacbb52)).

- Described [RK3576 boot flow](https://docs.flipper.net/one/resources/rockchip/boot-flow). Huge thanks to *@viktor-svirsky*.

## 🔌 Hardware

- Described the [Flipper One Debug Probe](https://docs.flipper.net/one/hardware/debug-probe).

  ![Debug Probe connected to Flipper One](/files/pics/dev-log/debug-probe-connection-to-flipper-one.png)

- To fix the problem of the battery cable disconnecting, we redesigned the connector so it's securely attached to the Main PCB with screws.

  ![New battery connector](/files/pics/dev-log/new-battery-connector.png)

- To simplify the device assembly process, we changed how the touchpad connects to the Main PCB. We replaced an FPC cable with a spring-loaded connector.

  ![Touchpad and haptic connectors](/files/pics/dev-log/touchpad-and-haptic-connectors.png)

- The old haptic motor was replaced with a more responsive one, which is also designed for a spring-loaded connection.

- Added an eSIM chip and a dual-SIM analog switch to enable switching between the physical SIM card and the eSIM from Linux.

  ![eSIM and dual SIM analog switch](/files/pics/dev-log/esim-and-dual-sim-analog-switch.png)

- The 3.3V output on the GPIO port is now separated from Flipper One's internal power rails, so a short circuit on the output won't affect the device.

- Replaced the backlight PWM driver to fix the issue where brightness below 5% caused the backlight to turn off.

- Reassigned the MCU BOOTSEL hotkeys to prevent accidental mode switching.

  ![BOOTSEL hotkeys](/files/pics/dev-log/hotkeys.png "New hotkey combination for switching the MCU to BOOTSEL mode")

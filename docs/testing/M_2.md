## Auto config detection

In the M.2 spec, **CONFIG pins** are **strap / detection pins** used by the **host** (and sometimes the module) to determine:

- What **interface** the module expects (USB vs PCIe)&#x20;
- What **role or mode** should it boot in
- sometimes the **module class** (WWAN, SSD, etc.)&#x20;

They are **sampled at power-up or reset**.

Depending on the **M.2 key**, different pins are used:

- **Key B / B+M (WWAN modems)**
  &#x20;Common CONFIG-related pins:&#x20;
  - `CONFIG_1`
  - `CONFIG_2`&#x20;
  - sometimes reused **GPIO / LED / UART pins**&#x20;
- **Key M (SSDs)**
  &#x20;Mostly PCIe / SATA focused, CONFIG pins usually select:&#x20;
  - PCIe vs SATA&#x20;
  - lane width (x2 / x4)&#x20;

&#x20;For **modems**, this is much more vendor-specific than for SSDs.

### ❓❓ How to test detect pins?❓❓

## M.2 modems&#x20;

![](https://api.archbee.com/api/optimize/3StCFqarJkJQZV-7N79yY/fQ5U0c1Dg1ydEB4AbfKoP_image.png)

M.2 modems can vary significantly:

- Different form factors. Flipper One supports only 30×52 and 30×42.&#x20;
- Different connection interfaces: USB and PCIe.&#x20;
- Different auxiliary pins: UART, LED, and other signals may be routed to unused pins of the M.2 connector.

### SIM-card test

1. Install SIM-card in Flipper One port&#x20;
2. Verify that it visible on modem

### USB-only modem&#x20;

1. Verify that modem only has USB interface&#x20;
2. Connect that it works properly

### PCIe-only modem&#x20;

1. Verify that modem only has PCIe interface (its exist?)
2. Connect that it works properly

### PCIe+USB modem

1. Verify that modem only has both PCIe+USB interfaces
2. Connect that it works properly

### Modem activity LED pins&#x20;

Most M.2 modems have activity LED on some pins.&#x20;
